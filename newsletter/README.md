# BH Monthly newsletter

A monthly digest of tax legislation moves + AI tooling updates relevant to Brookhaven's work. Published on the 1st of each month, auto-generated by Claude reading from curated sources.

## Architecture

```
sources.yml                                  ← trusted sources + priority topics
    │
    ▼
scripts/generate-newsletter.ts               ← Claude researches the web (API
    │                                            web_search tool) + drafts the issue
    ▼
web/src/content/newsletter/YYYY-MM.ts        ← typed issue file
web/src/content/newsletter/index.ts          ← updated to import it
    │
    ▼
git commit + push                            ← triggered manually OR by GitHub Actions
    │
    ▼
Netlify auto-build + deploy                  ← live on the site
```

**How the research works:** the generator uses the Anthropic Messages API
`web_search` tool — Claude searches the web live during generation, the same
way a person would, guided by the trusted-source list and priority topics in
`sources.yml`. It is NOT a fixed scraper; the source list is a starting point,
not a hard limit. Claude verifies the status of anything before reporting it
(e.g. won't say a bill "passed" unless a source confirms it).

## Setup (one-time)

### 1. Add ANTHROPIC_API_KEY to GitHub Actions secrets

In the repo on GitHub: **Settings → Secrets and variables → Actions → New repository secret**

- Name: `ANTHROPIC_API_KEY`
- Value: your Anthropic API key (starts with `sk-ant-...`)

That's all the workflow needs. Cost per monthly run is roughly **$3–6** (Claude does ~8-12 web searches plus token usage for reading results and writing the issue). Web search is billed per search (~$10 per 1,000 searches) on top of token costs — negligible at one run a month.

### 2. (Optional) Customize sources

Edit `sources.yml` to add, remove, or reprioritize sources. The generator handles any URL — it fetches the page, strips HTML to plain text, and feeds the result to Claude. No per-source code needed.

## How it runs

**Automatic:** the workflow at `.github/workflows/monthly-newsletter.yml` runs at **12:00 UTC on the 1st of each month** (7am Atlanta). It:

1. Checks out the repo
2. Installs the scripts dependencies
3. Runs `generate-newsletter.ts` (which fetches + drafts)
4. Commits the new `YYYY-MM.ts` issue + updated index
5. Pushes to `main`

Netlify sees the push and rebuilds the site, putting the new issue live within ~2 minutes.

**Manual** (from your machine):

```bash
cd scripts
npm install
ANTHROPIC_API_KEY=sk-ant-... npx tsx generate-newsletter.ts
# Or for a specific month:
ANTHROPIC_API_KEY=sk-ant-... npx tsx generate-newsletter.ts --month=2026-07
```

**Manual via GitHub Actions UI:** Actions tab → **Monthly Newsletter** → **Run workflow** → optionally specify a month.

## The two sections

Per the original spec:

1. **Tax legislation (primary)** — Federal + GA state moves that could affect filings. Status (Proposed / Markup / Passed / Effective / etc.), scope, and what it means for BH clients across brackets.
2. **AI tools (secondary)** — Claude / Cowork / Skills updates plus anything in the BH tool stack (Outlook, Monday, Dropbox, QBO, Teams). Less important.

If a section is genuinely quiet, the generator says so honestly rather than padding with fluff.

## Reviewing before publishing

Auto-generated newsletters land directly on `main` and deploy. If you want a human-review step:

**Option A: PR-based review.** Edit `.github/workflows/monthly-newsletter.yml` to push to a branch instead of `main`, and open a PR. You review, merge → deploy.

**Option B: Manual run + commit.** Don't set the cron. Run the generator locally on the 1st, review the output file, then commit and push yourself.

For BH, I'd start with **direct-to-main auto-publish** (every issue is improvable next month) and switch to PR review if a bad issue ever ships.

## Filtering hints in sources.yml

The `filters` section at the bottom of `sources.yml` passes hints to Claude:

- `lookback_days` — how far back to consider items (default: 35)
- `exclude_terms` — boilerplate to skip
- `prioritize_topics` — what BH cares about most (TCJA, SALT, estate tax, GA-specific items, etc.)

These are hints, not hard rules. Adjust as priorities shift.

## When the generator fails

Common causes:
- **`ANTHROPIC_API_KEY` not set** — set the secret, re-run from the Actions tab
- **Web search returned little** — quiet news month; the generator will honestly say a section is quiet rather than pad it
- **Claude output isn't valid JSON** — rare; the script extracts the outermost `{...}` and logs the raw text if it can't parse. Re-run; if it persists, tweak the system prompt in `generate-newsletter.ts`.
- **`web_search` tool rejected** — your Anthropic account/plan must have web search enabled. If the API returns a tool error, check the account's tool access.

## Adding a new section

To add e.g. "Industry news" as a third section:

1. Add a new array to `sources.yml` (e.g. `industry_news`)
2. Update `NewsletterIssue` in `web/src/content/newsletter/types.ts` to include the new section
3. Update `generate-newsletter.ts` to fetch + include the new section in the prompt + JSON output
4. Update `Newsletter.tsx` to render the new section

About an hour of work.
